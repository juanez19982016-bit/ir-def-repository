Server Actions

📘 Module 2 · Lesson 4⏱ 22:30📅 Updated March 2026
Overview
Resources
Comments (23)
Notes

Server Actions are asynchronous functions that are executed on the server. They can be used in Server and Client Components to handle form submissions and data mutations in Next.js applications.

What You'll Learn

  • How to define and use Server Actions
  • Form handling with useFormStatus and useActionState
  • Optimistic updates with useOptimistic
  • Error handling and validation with Zod
  • Revalidating data after mutations

Code Example

// app/actions.ts 'use server' import { z } from 'zod' import { prisma } from '@/lib/prisma' import { revalidatePath } from 'next/cache' const CreateProjectSchema = z.object({ name: z.string().min(1).max(100), description: z.string().max(500).optional(), }) export async function createProject(formData: FormData) { const data = CreateProjectSchema.parse({ name: formData.get('name'), description: formData.get('description'), }) await prisma.project.create({ data }) revalidatePath('/dashboard') }
